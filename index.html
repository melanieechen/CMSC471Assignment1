Directions:

Click and drag inside the map box to pan to different parts of the country. Use two fingers and drag to zoom in and out on the map. To restore the default view, click the "Reset Zoom" button above the map. The color grading allows you to visualize weather trends across the country. Use the dropdown menu to select which weather information you would like to visualize on the map.

Hovering on a weather station dot will allow you to see the name of the weather station, the state its located in, and the average values for temperature, precipitation, and snowfall recorded at that location.

Clicking on a weather station will cause a second graph to appear, displaying a line graph of weather measurements over time. Use the dropdown menus to select which weather information you'd like to visualize.

Note that for some weather stations, some types of data may not be available.

Assignment Write-Up

Introduction:
This project visualizes daily weather observations from the NOAA Global Historical Climatology Network (GHCN) dataset. The data was collected a various stations across the United States in 2017. The dataset provides geographic information (station name, state, latitude, longitude, elevation) and daily weather information such as:

  • TMIN – Minimum temperature (°F)
  • TMAX – Maximum temperature (°F)
  • TAVG – Average temperature (°F)
  • PRCP – Precipitation (inches)
  • SNOW – Snowfall (inches)
  • SNWD – Snow depth (inches)
  • AWND – Average wind speed (mph)
  • WSF5 – Fastest 5-second wind speed (mph)
  • WDF5 – Direction of fastest 5-second wind (degrees)
In our visualization, we focus on showing the distribution of stations across the US and their average recorded values for TAVG (average temperature), PRCP (precipitation), and SNOW (snowfall). We also allow users to explore trends for these values over time.

Rationale:
Since this data was already spatially distributed across the country, it felt natural to display it overlaid on a map of the United States. We also thought that mapping all the weather stations would help users find greater insights into how average temperature, precipitation, and snowfall are similar and different from region-to-region.

After plotting all the weather stations on the map, we noticed that many regions were densely populated so the dots for different stations overlapped. We decided to address this issue by adding a pan/zoom option so that users could take a closer look at those densely populated regions and explore individual stations they were interested in with more control. Because of the ability to infinitely pan, one of our developers (Anushka) got lost on the page and couldn't find her way back home to the map! Then she added a reset zoom button to help out any future users who might find themselves in a similarly sticky situation.

Next, we added a tooltips feature so that users could know which station was which. Along with the name of the station, we also decided to display the location's average recorded weather values when someone hovered on the point. This way, if someone wanted to get quick readings from a particular location, they could just hover and get that information.

We also noted that for many stations, there was data available from different timepoints. Just displaying the average value didn't seem like it did justice to this data available. Thus, we implemented the second graph to display the time-series data. We added a drop-down so that users can easily select the weather information they would like to investigate. We think it is interesting to be able to see how much a location's weather patterns vary (or don't vary) as the seasons change.

Following this, though we fulfilled all of our requirements, we realized that we weren't using the map visualization to its fullest potential. We decided to add color grading so that users could pan around the map and quickly see patterns without having to hover over every point. We tied this feature in with our dropdown menu, allowing users to select the weather information they would like to visualize via color grading. We considered using size as an indicator instead of color, but realized that the overlapping nature of the data points would make it difficult to rely on size. We decided color would be more efficient and visually appealing. We also added a legend to explain the color grading scale.
